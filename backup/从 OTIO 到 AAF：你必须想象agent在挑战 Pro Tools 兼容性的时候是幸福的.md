## 摘要

本文记录了一次 OTIO 转 AAF 的兼容性问题排查过程。问题表象是：开源库生成的 AAF 文件无法被 Pro Tools 正确读取，而 DaVinci Resolve 导出的文件则完全正常。排查过程中经历了工具链构建失败、无效的 Rust 重写尝试、结构对齐后的“假成功”，最终问题停留在一个无法解释的 30Hz 采样率异常上。

整个过程没有明确的失败节点，也没有收敛路径，更多体现为一种持续的技术性无效努力。

---

## 一、问题背景与初步假设

当前工作流无法输出 Pro Tools 可识别的 AAF 文件。对比测试显示：

* DaVinci Resolve 导出的 AAF：可正常导入
* 开源库生成的 AAF：无法打开

初步判断为 AAF 结构或字段存在差异，因此采用“对齐标准输出”的策略，通过结构对比逐步修正。

---

## 二、工具链构建与分析方法

### 2.1 AAF 分析工具的获取问题

为了分析 AAF 文件结构，需要使用官方工具进行解析。在 GitHub 上检索后发现缺乏可用的 Windows 预编译版本，因此改为自行构建。

过程中遇到的问题包括：

* GitHub Actions 配置错误，仅成功构建 Linux 版本
* Windows 构建流程卡死，排查耗时较长

在修正 CI 配置后，最终成功获取 Windows 版本工具 `dumpinfo`，用于结构化解析 AAF 文件。

---

### 2.2 一次无效的技术分支

在等待构建期间，尝试将原有 C++ AAF 库重写为 Rust 实现。该尝试由于缺乏充分设计与验证，很快失败，未产生可复用结果。

该分支未对主问题产生实质帮助。

---

### 2.3 基于 dumpinfo 的对齐策略

后续采用如下调试方法：

* 使用 `dumpinfo` 分别解析 DaVinci 输出与开源库输出
* 构建自动化对比流程，由 Agent 进行多轮比对与修正
* 逐步对齐字段、结构与引用关系

在多轮迭代后，两者的 dump 输出已高度一致，未发现明显结构性差异。

但 Pro Tools 仍然无法读取该文件。

---

## 三、逆向尝试与“假成功”

### 3.1 调试思路转向

在结构对齐无效的情况下，尝试从 Pro Tools 内部入手，在其加载 AAF 的路径上设置断点，以观察解析流程。

使用版本为 Pro Tools 12，并配合其源码进行调试。

---

### 3.2 异常现象

在准备调试时发现：

* Pro Tools 自带的 AAF 库
* 与官方源码编译版本

二者文件大小不一致。

尝试将自编译库替换原库后，出现意外结果：

> Pro Tools 可以成功加载开源库生成的 AAF 文件

---

### 3.3 数据层面的错误

虽然文件可以打开，但内容异常：

* 片段时间与位置正确
* 音频播放速度严重降低

进一步检查导入信息发现：

> 音频采样率被识别为 30 Hz

---

## 四、30Hz 问题与排查失败

### 4.1 参数验证

对所有相关字段进行排查：

* WAV Header：48000 Hz
* AAF WAVEDescriptor：48000 Hz
* Edit Rate：48000

未发现任何字段包含“30”这一数值。

---

### 4.2 问题特征

该问题具有以下特征：

* 无明确来源
* 不存在直接字段映射
* 无法通过结构对比发现

即：

> 错误存在于系统行为中，而非显式数据中

---

### 4.3 调试状态

尽管进行了大量比对与验证，该问题未能定位。理论上，通过进一步深入到二进制层或 Pro Tools 解析逻辑中，可能可以找到原因，但当前未继续推进。

---

## 五、技术总结

### 5.1 OTIO 音频时长的潜在误用

DaVinci Resolve 导出的 OTIO 文件中：

* `duration.rate` 表示视频帧率（如 24 或 30）
* 而非音频采样率

若直接用于采样计算，将导致音频速度异常。正确方式为：

1. 通过帧数与帧率计算时长
2. 再乘以音频采样率（48000）得到采样数

---

### 5.2 AAF Mob 写入顺序要求

Pro Tools 对 Mob 顺序存在隐性要求：

1. CompositionMob
2. MasterMob
3. SourceMob（TapeDescriptor）
4. SourceMob（WAVEDescriptor）

顺序错误可能导致解析失败，即使字段完全正确。

---

### 5.3 WAVEDescriptor 的隐含依赖

以下字段虽非规范强制，但对 Pro Tools 解析至关重要：

* 文件定位器（Locator）
* WAV 头摘要

缺失这些信息可能导致文件被拒绝加载。

---

## 结语

这次调试最终停在了一个无法解释的结果上：30Hz。

它不在任何已知字段中，也不在规范描述内，但确实影响了系统行为。问题没有被解决，路径也没有收敛。

从工程角度来看，这是一类典型的“可解但未解”的问题：理论上存在答案，但在当前资源与时间约束下，不具备继续推进的价值。

因此，它被保留在这里，作为一个未完成的状态。

问题依然存在。
